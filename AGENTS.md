# AGENTS

## Repository Scope
- CryptoBot Python is an unofficial client library for the Crypto Bot (Crypto Pay API) for Telegram.
- Keep this file aligned with `pyproject.toml`, `Makefile`, CI workflows, and the exported package API when project workflow changes.

## Repository Layout
- `cryptobot/__init__.py` exports the public clients: `CryptoBotClient` and `AsyncCryptoBotClient`.
- `cryptobot/_sync/client.py` contains the synchronous `httpx` client.
- `cryptobot/client.py` contains the async client, retry logic, and async pagination helpers.
- `cryptobot/models/__init__.py` defines dataclass DTOs and enums for API payloads.
- `cryptobot/webhook.py` provides signature verification, replay protection helpers, and the FastAPI listener.
- `cryptobot/_utils.py` and `cryptobot/errors.py` hold shared parsing helpers and exception types.
- `tests/test_client_mocked.py`, `tests/test_async_client.py`, `tests/test_webhook.py`, `tests/test_models.py`, `tests/test_utils.py`, and `tests/test_errors.py` cover unit behavior.
- `tests/test_sync_client.py` is the integration suite; it talks to Crypto Bot testnet when `--run-integration` is enabled and `API_TOKEN` is available.
- `docs/` uses Sphinx with MyST Markdown. `docs/cryptobot.rst` and `docs/modules.rst` are regenerated by `sphinx-apidoc`.
- Do not commit generated artifacts from `build/`, `dist/`, `htmlcov/`, `.pytest_cache/`, `docs/_build/`, or `__pycache__/`.

## Setup and Development Commands
- `uv sync` installs the default development environment.
- `uv sync --all-extras` matches the dependency set used in CI.
- `uv sync --extra webhook` installs FastAPI/Uvicorn extras for webhook work.
- `uv sync --extra docs` installs documentation tooling.
- `make lint` runs flake8, Ruff, and mypy.
- `make format` runs `ruff format` plus `ruff check --fix`.
- `make test` runs pytest with integration tests skipped by default.
- `make test-all` or `uv run pytest --run-integration` includes integration coverage.
- `make coverage` generates terminal, XML, and HTML coverage reports.
- `make docs` regenerates API reference stubs and builds the HTML docs.
- `make docs-translations` updates gettext catalogs; `make docs-es` builds the Spanish docs.
- `uv run uvicorn cryptobot.webhook:app --reload` is the quickest local webhook boot check.
- `make dist` builds wheel and source distributions; `make release` publishes them.

## Coding Conventions
- Target Python 3.10+ with 4-space indentation and a 127-character line limit.
- Use snake_case for modules, functions, and variables. Use PascalCase for dataclasses and enums that mirror Crypto Bot entities.
- Keep sync and async client APIs aligned when adding endpoints, validation rules, pagination, or error handling.
- Update public exports in `cryptobot/__init__.py` when introducing new public entrypoints.
- Prefer shared helpers and shared model parsing over duplicating request or response handling across clients.
- Use type hints consistently; `mypy` is part of the default lint target.

## Testing Expectations
- Put most new coverage in the mocked/unit suites unless the behavior genuinely requires the real API.
- Use `httpx.MockTransport`, patched `httpx.Client` or `httpx.AsyncClient`, and `httpx.ASGITransport` for isolated tests.
- Mark external-service coverage with `@pytest.mark.integration` and keep it optional by default.
- Integration tests rely on `API_TOKEN`; docs and examples typically use `CRYPTOBOT_API_TOKEN` or `CRYPTOBOT_TESTNET_TOKEN`.
- Add regression coverage beside the affected surface: sync client, async client, webhook, models, utils, or errors.
- Run `make lint` and `make test` before submitting changes. Run `make test-all` when modifying behavior that depends on the live API.

## Documentation and Release Workflow
- Update `README.md` and relevant docs pages when public APIs, examples, retries, or webhook behavior change.
- Rebuild docs after changing exported methods, signatures, or API-reference-visible docstrings.
- Follow Conventional Commits such as `feat(sync): ...`, `feat(async): ...`, `fix(webhook): ...`, `docs: ...`, or `test: ...`.
- Keep package version metadata aligned between `pyproject.toml` and `cryptobot/__init__.py`.
- GitHub Actions runs tests across Python 3.10 through 3.14; the publish workflow is tag-driven (`v*.*.*`).

## Security and Configuration
- Never hard-code secrets. Use `.env` or environment variables.
- Preserve webhook HMAC signature verification and replay-protection behavior when editing `cryptobot/webhook.py`.
- After upgrading `httpx`, `fastapi`, or `uvicorn`, rerun lint/tests and verify webhook startup locally.
